# A1 Anime Identity Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the persistent identity, profile, provider-refresh, request-approval, and pure-resolution foundation required by anime handling without changing acquisition, downloading, or import behavior.

**Architecture:** Keep movies and episodic TV in their existing pipelines. Add operator-owned profiles to `Movie` and `Series`, source-scoped aliases and episode coordinates under `Cinder.Catalog`, and a pure resolver that maps candidate coordinates to ordered stable episode IDs. TMDB remains the only metadata provider selected by A0; request profile proposals are confirmed at the existing approval gate.

**Tech Stack:** Elixir 1.20, Phoenix 1.8, LiveView 1.2, Ecto/SQLite, Req, ExUnit/Mox, Tailwind/daisyUI.

## Global Constraints

- Anime is a handling profile, not a third media type or pipeline.
- `episodes.id` remains canonical; provider coordinates never replace it.
- Mapping precedence is manual, then curated provider, then inferred provider.
- Provider refreshes replace only rows owned by that provider and namespace.
- Provider refreshes never overwrite manual aliases, manual classifications, `media_profile`, or already-returned resolver evidence.
- `:auto` remains effectively `:standard` without strong anime-provider identity; TMDB-only evidence may suggest Anime but cannot silently enable it.
- Requesters may propose Standard or Anime; manual approval explicitly confirms Standard or Anime before catalog creation.
- `auto_approve_all` accepts an explicit proposal, while no proposal preserves `:auto`.
- No A2 acquisition/search/parser/scorer work and no A3 durable mapping/import state lands in A1.
- Tests never use the network; TMDB behavior, implementation, and Mox mock change atomically.
- Every behavior change starts with a failing focused test, and the phase ends with `mix test` and `graphify update .`.

- [ ] **Step 4: Implement transactional source-scoped writes**

`Identity.replace_provider_aliases/5` and `replace_provider_coordinates/4` must delete only rows matching `{owner, source, namespace}` and `precedence != :manual`, then insert the replacement set. `Identity.put_coordinate/3` must load every episode through Season, reject a mixed series before inserting anything, and store positions with `Enum.with_index/1`.

The same-series write is one transaction:

```elixir
def put_coordinate(%Series{id: series_id}, attrs, episode_ids) do
  Repo.transaction(fn ->
    episodes =
      Repo.all(
        from e in Episode,
          join: season in assoc(e, :season),
          where: e.id in ^episode_ids,
          select: {e, season.series_id}
      )

    if length(episodes) != length(Enum.uniq(episode_ids)) or
         Enum.any?(episodes, fn {_episode, owner_id} -> owner_id != series_id end) do
      Repo.rollback(:episode_series_mismatch)
    end

    coordinate =
      %EpisodeCoordinate{series_id: series_id}
      |> EpisodeCoordinate.changeset(attrs)
      |> Repo.insert!()

    episodes_by_id = Map.new(episodes, fn {episode, _} -> {episode.id, episode} end)

    episode_ids
    |> Enum.with_index()
    |> Enum.each(fn {episode_id, position} ->
      %EpisodeCoordinateMembership{
        episode_coordinate_id: coordinate.id,
        episode_id: Map.fetch!(episodes_by_id, episode_id).id
      }
      |> EpisodeCoordinateMembership.changeset(%{position: position})
      |> Repo.insert!()
    end)

    Repo.preload(coordinate, memberships: [:episode])
  end)
end
```

`save_manual_alias/2` always sets `source: "manual"`, `namespace: "manual"`, and `precedence: :manual`. `update_manual_alias/3` and `delete_manual_alias/2` return `{:error, :not_manual_alias}` unless the row belongs to the supplied owner and has manual precedence; no caller can mutate a provider row by forging an ID.

- [ ] **Step 3: Implement the resolver**

```elixir
@precedence %{inferred: 0, curated: 1, manual: 2}

def resolve(coordinates, mappings, opts \\ []) do
  mappings = Keyword.get(opts, :overrides, []) ++ mappings
  matches = Enum.filter(mappings, &(&1.coordinate in coordinates and &1.episode_ids != []))

  case matches do
    [] -> maybe_ignore(opts)
    _ -> resolve_at_highest_precedence(matches)
  end
end
```

At the highest present precedence, preserve each mapping's episode order, de-duplicate identical ordered lists, and resolve only when one unique list remains. Conflicting lists return sorted candidates as `:ambiguous`. Evidence contains the chosen precedence and the matched coordinate/evidence records but never mutable schema structs.
